    • Trip Teddy

    A mobile app for travel itinerary planning and social trip sharing.

    TripTeddy 是一款用於旅遊規劃的應用程式，透過清晰的景點分類、個性化推薦行程清單和多人協作規劃機制，協助使用者在繁複的旅遊籌備過程中更輕鬆地發現並安排旅程。

    Overview

    Challenge :

    許多人在決定旅遊目的地、安排行程以及搜尋可信任的旅遊資訊時，常常遇到資訊過載與選擇困難。在編輯上很常使用表單或是文檔，難以彙整資訊。TripTeddy 提供精選的景點標籤與行程推薦，解決使用者在「找景點」、「行程規劃」、「群組協作」上的痛點，讓旅遊規劃更便利且樂趣十足。

    Goal :
    • 提升使用者的旅遊規劃效率：提供視覺化且易於理解的景點分類與行程建議，減少使用者搜尋與篩選信息的時間。
    • 打造差異化的「旅遊清單」功能：運用類似音樂播放清單的概念，讓使用者依據偏好（主題、地點、季節等）快速套用或編輯行程。
    • 促進使用者之間的協作與分享：透過多人協作規劃與一鍵分享功能，提升使用者黏著度。

    Research

    User Interview :

    為了深入了解使用者在旅遊規劃過程中遇到的問題，我們進行了使用者訪談，收集他們的需求、痛點與行為模式，從而提供有價值的設計方向。共分第一階段與第二階段 第一階段訪問有多元年齡層 有不同旅遊經驗的人（跟團、自助）第二階段 縮小為有豐富自助旅行經驗 並愛好自己規劃行程的用戶

    訪綱準備
  • 訪談問題聚焦於使用者的角色、規劃方式、主要痛點與期待的解決方案：
    1. 你通常在規劃旅遊時扮演什麼角色？
    2. 你平常如何規劃行程？
    3. 規劃旅遊時，遇到的最大困難是什麼？
    4. 在旅途中，你經常遇到哪些問題？
    訪談執行
  • 訪談問題聚焦於使用者的角色、規劃方式、主要痛點與期待的解決方案：
    1. 透過線上與面對面訪談，與不同類型的使用者交流，以獲取多樣化的意見。
    2. 記錄使用者回應，並標記重點引述以進一步分析。
    Empathy Map :

    根據 Say、Do、Think、Feel 四個面向的分析，我們可以整理出主要的 使用者需求與痛點，並提供相應的洞察與設計方向。

    User Pain Points :
    1. 行程規劃資訊分散，需在多個平台間切換
      • 使用者經常在 Google、YouTube、Instagram 搜尋旅遊資訊，規劃行程時還需搭配 Google Docs、Google Maps、Splitwise 等工具，整合資訊過於繁瑣。
    2. 行程協作不便，決策難以達成共識
      • 規劃過程中，使用者需要準備多個替代方案，以便團隊討論，但仍然難以滿足所有人的需求。
    3. 費用記錄與分帳困難
      • 旅途中通常由特定成員負責記帳，但手動計算與分帳容易遺漏或造成誤解。
    4. 缺乏視覺化行程概覽，資訊難以快速瀏覽
      • 文字訊息難以呈現多日行程概要，尤其是 Google Docs 外部連結較難閱讀。
    5. 旅行途中發生變動時，缺乏即時應變機制
      • 使用者擔心行程臨時變更，如天氣影響、部分成員無法參與，導致旅行體驗受影響。

    Problem Definition & Opportunities :

    POV #1：景點資訊過於分散，搜尋與選擇困難

    我們遇到 Zoan，他在搜尋與選擇旅遊景點時遇到了困難。

    我們發現… 網路上的資訊太過分散，使用者需要在不同平台間切換來蒐集資料，導致行程規劃變得麻煩且耗時。
    如果… 所有相關資訊都能夠匯總在同一個平台上，使用者可以一次性獲取所有旅遊景點的必要資訊，將會大幅提升行程規劃效率。

    HMW

    • HMW 如何標記每個景點的特點，讓資訊更清晰易讀？
    • HMW 如何提供現成的景點行程範本，幫助使用者更快決策？
    • HMW 如何讓景點選擇過程更有趣、更直覺？

    Solutions

    • ✅ 標籤化景點資訊，提供清晰分類（如：適合親子、熱門美食、自然風景等）。
    • ✅ 推薦行程範本，使用者可選擇適合自己的行程方案，或根據偏好修改。
    • ✅ 加入遊戲化互動，如「隨機推薦探索」、「旅遊挑戰任務」，讓選擇過程更有趣。

    POV #2：行程難以滿足所有參與者的需求

    我們遇到 Dennis，他在安排旅遊行程時很難滿足所有人的需求。

    我們發現… 旅遊決策通常由少數人負責，導致部分參與者需要妥協，影響旅遊體驗與滿意度。
    如果… 行程決策可以透過投票或其他協作方式，讓所有參與者的需求都能夠被考慮並整合，將能提升旅行的公平性與滿意度。

    HMW

    • HMW 如何找到讓所有人都滿意的行程？
    • HMW 如何讓大多數人接受替代方案，而不讓少數人感到不公平？
    • HMW 如何在考慮每個人的需求時，仍然保持行程規劃過程的簡單易用？

    Solutions

    • ✅ 投票系統，讓所有人參與行程選擇，確保每個人的需求被考慮。。
    • ✅ 自動推薦兼顧大多數需求的替代行程，確保少數人的需求也能獲得平衡。
    • ✅ 個人化偏好標記，讓使用者可以提前標註「最想去」的景點，系統自動匹配適合的行程。

    POV #3：旅遊費用分攤麻煩，經常遺漏開銷

    我們遇到 Wilson，他覺得記錄與分攤旅遊開銷過於麻煩。

    我們發現… 旅遊期間的開銷往往難以即時記錄，事後分攤費用時容易遺漏或造成混亂。
    如果… 開銷能夠根據行程自動記錄、計算並提供清晰的費用明細，將能讓費用管理變得更加輕鬆。

    HMW

    • HMW 如何幫助使用者記錄忘記的開銷？
    • HMW 如何讓開銷細節能夠清晰可見，確保每個人都清楚自己的支出？
    • HMW 如何讓費用分攤過程更加直覺、透明？

    Solutions

    • ✅ 標掃描發票與收據，自動記錄開銷。
    • ✅ 即時費用提醒，確保每筆開銷都能被記錄並分攤。。
    • ✅ 分帳工具與結算報告，讓使用者可以一鍵查看並公平分攤旅遊費用

    Summary :

    🔹 行程規劃的主要痛點：

    • 景點資訊過於分散，使用者需要在不同平台間切換，導致規劃困難。
    • 行程難以滿足所有參與者的需求，容易引發爭議或造成妥協。
    • 旅遊費用難以記錄與分攤，事後結算時經常遺漏或計算不清。

    🔹 解決方案核心概念：

    • 景點標籤化 + 旅遊推薦範本 → 簡化旅遊選擇過程，讓決策更直覺。
    • 投票機制 + 協作行程規劃 → 讓每位參與者都能夠參與決策，提升滿意度。
    • 自動化費用記錄與分帳功能 → 讓費用管理更透明、更方便。

    🔹 額外考量的設計要點：

    • 可視化行程總覽，讓使用者能夠快速瀏覽行程細節。
    • 社群功能整合，如 Instagram / Google 地圖資訊同步，提供更即時的旅遊推薦。
    • 動態行程調整功能，讓使用者能夠靈活應對突發狀況，確保旅途順利。
    User Needs & Opportunities :
    使用者需求 (Needs) 設計機會點 (Opportunities)

    幫助使用者快速達成行程共識 行程投票與多人協作功能，讓團隊成員共同決策，減少糾紛。

    自動化旅遊費用管理 內建費用記錄與智能分帳工具，自動計算與記錄旅遊開銷。

    Design

    Wireframe :

    根據研究得出的使用者需求與設計機會點進行APP功能的初步規劃與設計，整合行程規劃、協作為核心，透過可視化時間軸、熱門行程選單，簡化規劃流程並提升使用體驗。

    • 根據初步設計選出最直觀的選單與介面。
    • 確認介面都具備完整的流程與功能，並確認主要按鈕的易用性。
    • 在重要功能中，除了文字之外，加入用圖片，以幫助使用者想像和理解內容。
    • 整理出時間軸是使用者希望納入應用中的重要功能
    Low-fi prototype structure :

    進一步優化Wireframe，並根據Wireframe所歸納的主要功能設計Low-fi Prototype，Low-fi Prototype 著重於提升互動體驗與優化使用流程，確保行程規劃直覺且流暢。

    • 首頁行程總覽：日程卡片 + 地圖標示，快速掌握每日計畫。
    • 探索頁面：標籤分類、篩選功能、隨機推薦、旅遊挑戰。
    • 行程規劃：拖曳調整順序、投票機制、留言討論，提高決策效率
    • 費用管理：掃描發票、自動記錄、確認機制，確保分帳透明
    • 動態調整：即時推播通知 + 替代方案建議，快速應變變動行程
    Medium-Fi Prototype :

    Medium-Fi Prototype 進一步優化使用者體驗，畫出所有流程，設計整體風格，針對介面易用性、資訊呈現與互動方式進行調整，以確保行程規劃更加直觀且高效

    • 設計整體風格：統一視覺元素，提升介面一致性與可讀性。
    • 繪製完整操作流程：確保頁面之間的邏輯順暢，提升整體使用體驗。
    • 優化 UI 圖示與命名：避免使用者混淆，提高易用性與直覺性。
    • 彈出視窗整合與改進：確保功能清晰易用，減少重複與資訊分散。
    • 強化視覺資訊呈現：提供更多關鍵資訊，提升行程決策效率。

    Testing

    User Testing & Iterations :

    為進一步優化 Medium-Fi Prototype，Usability Testing 觀察使用者行為，並以「Think Aloud」收集即時回饋。測試任務涵蓋尋找符合需求的行程、依範本規劃新行程、分享行程結構，以驗證系統的易用性與操作流暢度。

    測試觀察與發現
    • 「Clone」一詞不直覺，部分使用者無法理解其功能。
    • 使用者希望能更直接完成目標，避免重複步驟。
    • 點擊習慣因人而異，需提升設計的包容性。
    迭代與優化
    • 「用語調整：以更直白、易理解的詞彙取代模糊術語，提升可讀性。。
    • 流程精簡：減少冗長步驟，使操作更直覺順暢。
    • 互動優化：重新設計圖示與按鈕，提高包容性與點擊辨識度。
